Features

Why Use Debate?

Find weaknesses in arguments before someone else does.

Thesis Construction

Build well-structured arguments with clear premises, conclusions, and confidence levels.

Antithesis Generation

Systematically challenge your position with opposing viewpoints and counter-arguments.

Synthesis Integration

Combine insights from both sides into stronger, more nuanced conclusions.

Argument Tracking

Track relationships between arguments - what supports, contradicts, or responds to what.

Strength Analysis

Identify and document both strengths and weaknesses in each argument.

Confidence Scoring

Assign and track confidence levels as arguments evolve through the debate.

See It in Action

A remote work debate with structured argumentation

Thesis

Remote-first companies outperform hybrid companies in employee productivity and retention

75%
Confidence

Premises

1

Studies show remote workers report 13% higher productivity

2

Remote companies have access to global talent pool

3

Employees save 2+ hours daily on commute

Conclusion

Companies should adopt remote-first policies to maximize productivity

Strengths

+ Backed by productivity studies
+ Proven by GitLab, Zapier

Weaknesses

- Not all industries applicable
- Junior mentorship concerns
Input Thesis 
{
  "claim": "Remote-first outperforms hybrid",
  "premises": [
    "13% higher productivity",
    "Global talent pool access",
    "2+ hours saved on commute"
  ],
  "argumentType": "thesis",
  "confidence": 0.75,
  "nextArgumentNeeded": true
}
Output 
{
  "argumentId": "thesis-001",
  "strengths": [
    "Backed by studies",
    "Proven by GitLab"
  ],
  "weaknesses": [
    "Not all industries",
    "Mentorship concerns"
  ],
  "suggestedNextTypes": ["antithesis"],
  "nextArgumentNeeded": true
}

Reference

API Parameters

Parameter Type Required Description
claim string Required Main assertion being made
premises string[] Required Supporting reasons for the claim
argumentType enum Required thesis | antithesis | synthesis | objection | rebuttal
confidence number Required 0-1 belief level in the argument
nextArgumentNeeded boolean Required Whether another argument is needed
respondsTo string Optional ID of argument being responded to
contradicts string[] Optional IDs of arguments this opposes
